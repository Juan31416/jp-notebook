# Incremental Static Regeneration (ISR)

▶️ _allows content editors & developers to use static generation on a per-page basis & udpate content without having to rebuild the entire site._

A way building an static site with thousands of pages with the next features:

- Update content in a matter of seconds.
- Without the need of rebuild or redeploy the whole site.
- Design for more than a cache.

Pros:

- None waiting time for content to update.
- None large compilation time.

Cons:

- For "realtime" application you don't want to make users wait.
- Server redering will make the user wait.
- For really small sites.
